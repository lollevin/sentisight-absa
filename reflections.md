# Reflections on Learnings

This project bridged AI and marketing — two fields I previously saw as separate. The key insight: "sentiment analysis" isn't just labeling reviews good or bad; real value lies in extracting *actionable* aspect-level insights.

Building SentiSight taught me three things. First, prompt engineering is critical — the difference between generic LLM output and structured JSON comes down to system prompt design. Small changes like requiring exact quotes dramatically improved quality. Second, evaluation is non-negotiable. Building a labeled dataset and comparing ABSA against VADER with precision/recall/F1 gave concrete evidence the approach works — not just intuition. Third, good UX makes AI accessible. Bilingual support, theme switching, and friendly error messages transformed a prototype into something usable by real marketers.

The biggest growth came from iterative debugging — handling API failures, fixing CSS across themes, and optimizing with caching. These taught me systematic problem-solving I'll carry forward.
